I thought it would be cool to try and make an image/content slider using only HTML, and css. I was hoping to make it completely navigate-able (I think that's a word) by using the keyboard. FYI I am writing this as I am figuring it out, so there will probably be some trial and error here.

That said, my first step is to see how tab index and the keys work with a pretty plain set of radio buttons. 

<div class="radio-slider-container">
  <input type="radio" name="radio" value="" id="radio1">
  <label for="radio1">Label 1</label>
  <input type="radio" name="radio" value="" id="radio2">
  <label for="radio2">Label 2</label>
  <input type="radio" name="radio" value="" id="radio3">
  <label for="radio3">Label 3</label>
</div>

If you click the paragraph before the examples, and hit tab, you bring focus to the radio buttons. The right and left arrow keys then work to select different options. The up and down arrow keys work too.

Now that I now you can change the value with the arrow keys, I need to think about how to make this into a slider. While the arrow keys work to change the radio button selected, I want arrows on the left and right that work as well. To me this sounds like I need 2 labels per button. Like this. 

<div class="radio-slider-container">
  <input type="radio" name="radio2" value="" id="radio10">
  <label for="radio10">First Label 1</label>
  <label for="radio10">Second Label 1</label>
  <input type="radio" name="radio2" value="" id="radio20">
  <label for="radio20">First Label 2</label>
  <label for="radio20">Second Label 2</label>
  <input type="radio" name="radio2" value="" id="radio30">
  <label for="radio30">First Label 3</label>
  <label for="radio30">Second Label 3</label>
</div>

Both of the labels work for each radio button. I just checked, and that validates to. I was a little concerned it wouldn't like 2 labels for one thing, so that's good. If you're wondering, the reason I id's and names is so that this is independent from the first example.

So here's my thought on how I will lay this out. The first row represents the slides, and the second row represents the labels for the slide. Other than the first and last slide, each slide has the 2 labels. I think for the first and last slide the second labels would just be out of view. The labels will be overlayed over the slides aligned exactly like below, so if you click the left of a slide you will go to the previous slide (if not on the first), and if you click the right side you will go to the next slide (if not on the last).

A good next step will be to create the slider with just a scroll bar on the bottom. I am going to use a couple of containers, relative and absolute position, and flexbox to do this. Found some sweet pictures on the internet to test with. My goal for now is to get this working, so I can make this easier by using images that are all the same size. I'll eventually get this thing to work with different 'slide' aspect ratios. Anyway, back to the thing.

And the scss... 

.slider-container {
  width: 500px;
  overflow-x: scroll;
  margin: 0 auto;

  .slider-container-flex {
    position: relative;
    display: flex;
    width: 1500px;

    .slide {
      width: 500px;
    }
  }
}

What I will want to do is make .slider-container overflow-x: hidden;. Then, you will only be able to see one image at a time, and won't be able to scroll. What I should be able to do, is then move the reletively positioned .slider-container-flex left and right depending on what radio button is selected. The tricky part about this will be making the same html and css work with any number of slides. Again, for now I just want to get this working, so I'll come back to that at some later point.
